# 使い方（校正・読み取り・自律・故障復帰）

前提：[install.md](install.md) で書込み・配線・ライブラリ導入まで済んでいること。
API の網羅は [api.md](api.md)、厳密仕様は [docs/SPEC.md](../SPEC.md)。

---

## 1. 基本の読み取り

```cpp
#include <Wire.h>
#include <WaterBoader.h>

WaterBoader wb(0x20);          // I2C アドレス 0x20

void setup() {
  Serial.begin(115200);
  Wire.begin();
  Wire.setClock(100000);       // 100kHz
  wb.begin(/*avrPullups=*/true); // クラシックAVRは true で A4/A5 内部プルアップも有効化
}

void loop() {
  int w = wb.water();          // 1=水あり, 0=水なし, -1=I2Cエラー
  long v = wb.value();         // 生の差動値（水で下がる、20000中心）
  if (w == 1) {
    // 水あり時の処理（警報・ポンプ停止・ヒーター空焚き防止など）
  }
  delay(500);
}
```

- `water()` は **PIC 側でヒステリシス判定した最終結果**。ホスト側でしきい値計算は不要。
- `-1`（エラー）は I2C の一時失敗。連続する場合は「故障復帰」（後述）。

## 2. 校正ウィザード（据付時に一度）

静電容量式は取り付け環境で値が変わるため、**現場で一度校正**します。
`calibrate()` は 3 状態を対話式に取得します。

```cpp
WbConfig cfg;
if (wb.calibrate(cfg)) {       // 空気 → 乾いたガラス → 水、各50サンプル
  if (wb.pushConfig(cfg) && wb.save()) {
    Serial.println("EEPROMへ保存完了");
  }
} else {
  Serial.println("校正失敗：乾と水が十分に分離していません");
}
```

### ウィザードの流れ（シリアルの指示に従う）
1. **空気中**：電極に何も触れていない状態で Enter → 50 サンプル取得
2. **乾いたガラス**：電極を乾いたガラス壁に当てて Enter → 50 サンプル
3. **水**：ガラスに水を入れた（or 水位が電極位置に来た）状態で Enter → 50 サンプル

ウィザードが自動計算するもの：
- `wbound`（ON しきい値）= **乾と水の中点**
- `hyst`（ヒステリシス幅）= **乾のばらつき σ の 0.7 倍**（最低2）
- OFF しきい値 = `wbound + hyst`

> **分離が不十分だと失敗**します（`乾 ≤ 水` や `分離 < 4σ` の警告）。その場合は
> 電極の接触長を伸ばす／ガラスを薄くする／[hardware.md](hardware.md) の電極改善を検討。

### 判定ロジック（PIC 内部）
```
水なし状態: value <= wbound            なら「水あり」へ
水あり状態: value <  wbound + hyst      の間は「水あり」を維持、超えたら「水なし」へ
```

## 3. EEPROM 自律（ホスト不在でも動く）

`save()` で校正値は **PIC 自身の EEPROM** に書かれ、**毎回の電源投入時に自動復元**されます。

```cpp
if (wb.configValid() == 1) {
  // PIC が EEPROM から校正値を復元済み → そのまま使える
}
```

- これにより、**ホスト(Arduino/ESP32)が落ちても・繋がっていなくても、PIC 単独で water/dry を判定し続けます**。
  システム全体の冗長化になります。
- 値空間が変わるファームを焼くと EEPROM の magic が変わり、旧設定は自動的に無効化
  （`configValid()==0`）→ 再校正を促します。

## 4. 故障復帰（I2C エラーからの自己回復）

```cpp
static uint8_t miss = 0;

void loop() {
  long v = wb.value();
  if (v < 0) {                 // I2C 失敗
    if (++miss >= 5) {
      wb.recover();            // PIC をリセット→WDT復帰を待つ→リセット要因を表示
      miss = 0;
      wb.getConfig(cfg);       // 復帰後に設定を読み直す
    }
  } else {
    miss = 0;
    // 通常処理
  }
  delay(500);
}
```

- ライブラリは各 I2C 操作を内部で `WB_RETRY`(=3) 回まで再試行します。
- `recover()` は PIC をソフトリセットし、ウォッチドッグリセットの復帰(約1秒)を待ってから、
  **なぜ止まったか**（`resetCause()`）を表示します。
- リセット要因ビット：`POR / BOR(電圧サグ) / WDT(暴走を自動復帰) / MCLR / soft-RESET`。
  `WDT` が立っていれば「ファームが一度ハングして自動復帰した」の意。

## 5. デバッグ用の生値観測（差動の効きを確認）

`OutSel`(0x21) を切り替えると、通常の差動値の代わりに **センス生値 / リファレンス生値**を読めます。
差動でノイズが相殺されているかの確認に使います。

```cpp
wb.setOutSel(1);  // 1 = センス生値
long s = wb.value();
wb.setOutSel(2);  // 2 = リファレンス生値
long r = wb.value();
wb.setOutSel(0);  // 0 = 差動(通常運用) に戻す
```

- RF を電極に近づけたとき、**センス/リファレンス両方が動くのに差動値はほぼ不動**なら、
  共通モード除去（＝EMI 対策）が効いています。
- モードを切り替えたらホスト側の基準がずれるので、観測後は `0`（差動）に戻すこと。

## 6. 商用電源ノイズ（50/60Hz）対策：ゲート長

`Gate`(0x20) は測定の積分時間（約100µs単位）。**電源1周期に合わせると商用ハムを打ち消せます**。

- **50Hz 地域**：`gate=200`（約20ms＝50Hzの1周期）… 既定
- **60Hz 地域**：`gate≈167`（約16.7ms）

```cpp
wb.setGate(167);   // 60Hz 地域向け（必要なら）
```

> 校正はゲートを決めてから行ってください（ゲートを変えると値が変わります）。

## 7. 温度の読み取り（副次機能）

```cpp
wb.setTempCal(4.8, 4.0, 94);   // 5V 版の既定（3.3V は下記）
float t = wb.tempC();          // ℃（エラー時 NAN）
```

- ダイ温度は 1 点校正で **±1℃程度**。過熱警告向けで、計測器グレードではありません。
- **3.3V 版**：ファームを 3.3V 用（低レンジ）で焼き、`setTempCal(3.3, 2.0, N)` の `N` を
  既知温度で再導出（[api.md](api.md) 参照）。未校正だと表示が数百℃になります（異常ではなく未校正）。
