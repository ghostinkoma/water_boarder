## ピン割り当て（PIC12F1840・8ピン）

| ピン | 名称 | 役割 | 兼用の注意 |
|---|---|---|---|
| 1 | VDD | 電源（5V または 3.3V） | ホストと同一電圧にする |
| 2 | RA5 | 動作表示 LED（任意） | 生存確認に便利 |
| 3 | **RA4** | **センス電極（CPS3）** | ICSP と干渉しない＝電極に最適 |
| 4 | RA3 | MCLR / VPP（書込み） | ICSP |
| 5 | **RA2** | **SDA（I2C データ）** | 固定 |
| 6 | **RA1** | **SCL（I2C クロック）** | **ICSPCLK 兼用**→書込み後 PICkit を抜く |
| 7 | **RA0** | **リファレンス電極（CPS0）** | **ICSPDAT 兼用**→書込み時はジャンパで切離す |
| 8 | VSS | GND | ホストと共通 |

## 配線図（最小構成）

```
   ホスト(Arduino/ESP32)               PIC12F1840
   ┌────────────────┐               ┌───────────────┐
   │ VDD ───────────┼───┬───────────┼─ 1 VDD         │
   │                │  ═╪═ 1µF+100nF │    (VDD-VSS間) │
   │ GND ───────────┼───┼───┬───────┼─ 8 VSS         │
   │ SDA(A4/GPIO) ──┼─┬─┼───┼───────┼─ 5 RA2 (SDA)   │
   │ SCL(A5/GPIO) ──┼┬┼─┼───┼───────┼─ 6 RA1 (SCL)   │
   └────────────────┘││ │   │       │                │
          ┌──────────┘│ │   │       │  3 RA4 ────────┼── センス電極
     4.7k ┤(→VDD)      │ │   │       │  7 RA0 ────────┼── リファレンス電極
     4.7k ┤(→VDD)──────┘ │   │       │  2 RA5 ────────┼── LED(任意)→GND
          └──────────────┘   └───────┴────────────────┘
                                   共通GND
```

要点：
- **4.7kΩ プルアップ**を SDA/SCL に、ホスト VDD へ。I2C の MISS 対策として実務上ほぼ必須。
- **GND 共通**、**電源電圧をホストと一致**（3.3V/5V 混在は MISS 多発）。
- デカップリング **1µF + 100nF** を PIC の電源直近に。

## 電極の作り方（性能を左右する最重要部）

実測で確認された基準構成：

- **材質**：AWG24 単線
- **形状**：**ガラス面に約2cm 接触**させる（センス電極 = RA4）
- **差動化**：リファレンス電極（RA0）を **センス電極の背中合わせ**に配置。
  水はセンス側だけに効き、ノイズ（温度/湿度/電波/電源変動）は両方に等しく乗って相殺される。
- **引き回し**：電極リードは**短く、SDA/SCL から離す**。引き回しが変わると容量が変わるので固定する。
- **RA0 は ICSPDAT 兼用**：書込み時にリファレンス電極を切り離せるよう**ジャンパ**を入れておく。

> ⚠ **ブレッドボード不可**：接点容量が pF 級の信号を埋もれさせる。必ずハンダ付けした基板で。

### さらに性能を上げたい場合（任意）
裸の単線は最小構成です。以下で **マージン・EMI耐性・再現性**が上がります（誤検出率自体は既にほぼ0なので、
主に「その0を条件が悪くても保つ」方向の改善）：
- **平面電極**（銅箔/PCBパッド）で結合面積を増やす → 水信号が大きくなる
- **ガードリング**（できれば駆動ガード）で環境への漂遊容量を抑え、EMI/手の影響を減らす
- **同軸/シールド線＋フェライト**で開放電極が拾う商用ハム/RFを遮蔽
- **剛体で電極↔壁の距離を固定**してドリフト源を除去

> 面積を増やすときは**ガードとセット**にすること（大きい電極はアンテナにもなるため）。
> 電極を変えたら**必ず再校正**（値空間が変わる）。

## 電源電圧（5V / 3.3V）

- 使う電圧に合ったファームを書く：`i2c_wb_5V`（高レンジ温度）/ `i2c_wb_3V3`（低レンジ温度）。
  高レンジ温度は VDD≥3.6V が必要なので **3.3V は必ず低レンジ版**。
- 水位センシングは**どちらの電圧でも動作**（3.3V 実測でも乾/水ギャップ ~81、5V の ~88 と同等）。
  ただし**動作電圧で境界を再校正**すること。
- ホスト側の温度校正も電圧に合わせる（[api.md](api.md) の `setTempCal` 参照）。
