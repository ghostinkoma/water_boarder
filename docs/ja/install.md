# 構築ガイド（初心者向け・これ1枚で最後まで）

このページだけを順番に進めれば、WaterBoader を **ゼロから動かす**ところまで到達できます。
専門用語には最初に一言説明を付けています。分からない単語が出たら
[トラブルシューティング](troubleshooting.md) と [用語のミニ解説](#用語のミニ解説) も参照してください。

> 全体像：**PIC(センサ) にファームを書き込み → ホスト(Arduino/ESP32) にライブラリを入れ →
> 2枚を I2C で繋ぎ → 一度だけ校正**。この4ステップです。

---

## 0. 必要なもの（部品と道具）

### 部品（ハードウェア）
| 品目 | 用途 | 補足 |
|---|---|---|
| **PIC12F1840**（DIP または SOP） | センサ本体のマイコン | 8ピン。これがセンサの頭脳 |
| ホストマイコン | 値を読む側 | **Arduino Uno/Nano(5V)** か **ESP32(3.3V)** など。I2C(Wire) が使えれば可 |
| 電極用の線 **AWG24 単線** ×2 | センス電極 / リファレンス電極 | 実測構成は「ガラス面に約2cm 接触」 |
| 抵抗 **4.7kΩ** ×2 | I2C プルアップ | SDA / SCL 用。ホスト側に付ける |
| コンデンサ 1µF + 100nF | 電源デカップリング | PIC の VDD-VSS 間 |
| LED + 抵抗（任意） | 動作表示 | PIC の RA5(pin2) に付けると生存確認できる |
| 小型基板 / ユニバーサル基板 | 実装 | **ブレッドボード不可**（後述） |
| 水槽壁の代用：**ガラス瓶** と 少量の水 | 動作確認 | 実測は「約20ccの水・ただのガラス瓶」 |

> ⚠ **ブレッドボードは使えません。** 静電容量式は pF レベルの微小容量を見るため、
> ブレッドボードの接点容量に信号が埋もれます。必ずハンダ付けした基板で。

### 道具（プログラマとソフト）
| 品目 | 用途 |
|---|---|
| **PICkit 3/4**（PIC プログラマ） | PIC にファームを書き込む機材 |
| **MPLAB X IDE**（無料） | PIC の開発環境。ビルドと書込に使う |
| **XC8 コンパイラ**（無料） | PIC 用 C コンパイラ |
| **DFP（Device Family Pack）** `PIC12-16F1xxx_DFP` | MPLAB X に PIC12F1840 を認識させる部品 |
| **Arduino IDE** または **PlatformIO** | ホスト側のライブラリを書き込む |
| USB シリアル（ホストの USB） | 校正ウィザードと表示に使う（115200bps） |

## 1. PIC 開発ツールを入れる

1. **MPLAB X IDE** を Microchip 公式から入れる（無料）。
2. **XC8 コンパイラ** を入れる（無料。本プロジェクトは v4.00 で確認済み）。
3. **DFP** `PIC12-16F1xxx_DFP`（1.9.258 で確認済み）を MPLAB X の *Tools → Packs* から入れる。
   これが無いと PIC12F1840 がデバイス一覧に出ません。

> これで「PIC12F1840 を選んでビルドできる」状態になります。

---

## 2. PIC にファームを書き込む（センサを作る）

### 2-1. 電圧に合ったファームを選ぶ
- ホストが **5V 系（Arduino Uno 等）** → `firmware/i2c_wb_5V/i2c_wb_5V.X`
- ホストが **3.3V 系（ESP32 等）** → `firmware/i2c_wb_3V3/i2c_wb_3V3.X`

> **ホストと PIC は必ず同じ電源電圧**にします（3.3V と 5V の混在は I2C エラーの原因）。
> 2つのファームの違いは温度レンジ1行（`TEMP_HIGH_RANGE`）だけです。

### 2-2. MPLAB X で開いてビルド
1. MPLAB X で *File → Open Project* から上記 `.X` フォルダを開く。
2. ハンマー（Build）アイコンでビルド。`PIC12F1840` 向けにコンパイルされます
   （Program 約 1540 words / 37.6%、RAM 164B / 64% なら正常）。

### 2-3. PICkit で書き込む（配線に注意）
ICSP（書込み）配線は **5本**：

| PIC ピン | ICSP 信号 |
|---|---|
| 1 VDD | 電源 |
| 8 VSS | GND |
| 4 RA3 | **MCLR / VPP** |
| 7 RA0 | **ICSPDAT**（＝リファレンス電極と兼用！） |
| 6 RA1 | **ICSPCLK**（＝SCL と兼用！） |

> ⚠ **書込み時の重要注意**
> - **RA0 はリファレンス電極 と ICSPDAT の兼用**。書込み中は**リファレンス電極をジャンパで切り離す**と安全。
> - **RA1 は SCL と ICSPCLK の兼用**。**書込み後は PICkit を抜く**（挿しっぱなしだと I2C を邪魔します）。
> - 書込みが不安定なら、まず I2C プルアップ/電極を一時的に外して切り分ける（配線かコードかを一瞬で分けられる）。

書込み後、RA5(pin2) の LED が点滅していればファームは動いています。

## 3. 配線する（2枚を I2C で繋ぐ）

最小配線（詳細と電極の作り方は [hardware.md](hardware.md)）：

```
   ホスト(Arduino/ESP32)              PIC12F1840
   ┌───────────────┐              ┌──────────────┐
   │ VDD(5V/3.3V) ──┼───┬──────────┼─ pin1 VDD    │
   │ GND ───────────┼───┼───┬──────┼─ pin8 VSS    │
   │ SDA (A4/GPIO) ─┼─┬─┼───┼──────┼─ pin5 RA2 SDA│
   │ SCL (A5/GPIO) ─┼┬┼─┼───┼──────┼─ pin6 RA1 SCL│
   └───────────────┘││ │   │      │              │
        4.7k×2 ──────┘│ │   │      │  pin3 RA4 ───┼─ センス電極 (水槽壁に貼る)
        （SDA/SCLを    └─┘   │      │  pin7 RA0 ───┼─ リファレンス電極 (背中合わせ)
         VDD へプルアップ）    │      └──────────────┘
                              └ 共通GND
```

- **4.7kΩ プルアップ**を SDA と SCL に、ホストの VDD へ向けて付ける。
- **GND は必ず共通**に。
- 電極 2 本は **背中合わせ**に配置（差動でノイズを打ち消すため）。

---

## 4. ホストにライブラリを入れて動かす

### 4-1. ライブラリを導入
- **Arduino IDE**：この repo の `src/` を `WaterBoader` というフォルダ名にして
  `Arduino/libraries/WaterBoader/` に置く（中に `WaterBoader.h/.cpp`）。
  または repo 全体を ZIP にして *スケッチ → ライブラリをインクルード → .ZIP形式のライブラリをインストール*。
- **PlatformIO**：`platformio.ini` の `lib_deps` にこの repo を追加
  （例：`lib_deps = https://github.com/ghostinkoma/water_boarder.git`）、
  またはローカル参照（`lib_deps = symlink://../WaterBoader`）。

### 4-2. サンプルを書き込む
`examples/WaterBoaderDemo/WaterBoaderDemo.ino` を開いてホストに書込み、
シリアルモニタを **115200bps** で開きます。

### 4-3. 初回起動（校正するか聞かれる）
起動すると次のように表示されます：

```
=== WaterBoader demo ===
reset cause: ...
stored calibration: none            ← まだ校正していない
C = calibrate / S = start.  (10s timeout = start)
```

- **`C` を押す** → 校正ウィザードが始まる（次章）。
- 何もしないと 10 秒で測定モードに入ります（未校正だと判定は不正確）。

---

## 5. 一度だけ校正する（据付現場で）

`C` を押すと 3 状態を順に取ります（詳細は [usage.md](usage.md)）：

1. **空気中**（電極に何も無い状態）で 50 サンプル
2. **乾いたガラス**に電極を当てて 50 サンプル
3. **水を入れたガラス**ごしで 50 サンプル

ウィザードが自動で `wbound`（しきい値）と `hyst`（ヒステリシス幅）を計算し、
`pushConfig()` → `save()` で **PIC の EEPROM に保存**します。以後：

```
saved to EEPROM.
readback: gate=200 wbound=... hyst=... valid=1
```

`valid=1` になれば成功。**次回からは電源投入だけで PIC が校正値を自動復元**し、
ホスト無しでも単独判定できます。

---

## 6. 動作確認

測定モードに入ると 0.5 秒ごとに 1 行出ます：

```
t(s)  value  water  sense  ref  tempC
0     19290  WATER  2240   2939  ...
1     20422  dry    ...
```

- 電極に**水を近づけると `value` が下がり `WATER`**、離すと `dry` に戻れば成功です。
- `MISS` が出る場合は I2C 配線/プルアップ/電圧を確認 → [troubleshooting.md](troubleshooting.md)。

🎉 ここまで来れば構築完了です。API の詳細は [api.md](api.md)、使いこなしは [usage.md](usage.md) へ。

## 用語のミニ解説

- **I2C**：2本の線(SDA/SCL)で複数機器を繋ぐ通信。WaterBoader は「スレーブ(応答役)」で住所は `0x20`。
- **CPS（容量センシング）**：PIC 内蔵の、電極容量を周波数に変えて測る回路。水が近づくと容量が増え、値が下がる。
- **差動 / リファレンス電極**：2電極の差を取ってノイズ(温度/湿度/電波)を打ち消す方式。
- **ヒステリシス**：ON と OFF のしきい値をずらして、境界での「バタつき(チャタリング)」を防ぐ仕組み。
- **ICSP**：PIC に回路上のまま書き込む方式。RA0/RA1/RA3 を使う（電極/ I2C と兼用なので注意）。
- **EEPROM 自律**：校正値を PIC 自身に保存し、ホストが居なくても判定を続けられること。
- **ウォッチドッグ(WDT)**：暴走を検知して自動リセットする安全機能。
