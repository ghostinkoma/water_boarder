# トラブルシューティング（症状 → 対処）

実機で確定した知見に基づく対処集です。困ったらまずここ。

---

## 通信できない / `MISS`・`-1` が続く

I2C の読み取りが失敗している状態です。上から順に確認：

1. **プルアップ抵抗**：SDA/SCL に **4.7kΩ をホスト VDD へ**付けたか。無い/弱いと MISS が多発。
   → 実務上いちばん多い原因。ホスト側で明示プルアップ（`begin(true)` の A4/A5 内部プルアップ）＋外付け4.7kを推奨。
2. **電源電圧の一致**：ホストと PIC が**同じ電圧**か（3.3V と 5V の混在は MISS 多発）。
3. **GND 共通**：両者の GND が繋がっているか。
4. **PICkit を抜いたか**：**RA1=SCL は ICSPCLK 兼用**。書込み機材を挿しっぱなしだと I2C を邪魔する。
5. **アドレス**：既定 `0x20`。ホストで I2C スキャンして 0x20 が見えるか。
6. **クロック**：`Wire.setClock(100000)`（100kHz）にしているか。
7. それでもダメなら `wb.recover()` で PIC をリセットし、`resetCause()` を確認。

---

## 書き込み（プログラミング）が失敗する / 不安定

- **RA0（ICSPDAT）はリファレンス電極と兼用**。書込み時は**リファレンス電極をジャンパで切り離す**。
- **RA1（ICSPCLK）は SCL と兼用**。I2C プルアップやバスが書込みを乱すことがある→**ジャンパで一時切離し**して切り分け。
- 配線かコードかを迷ったら、まず**電極・I2Cバスを外して最小構成**で書込みを試す。
- ソースは MPLAB X エディタ上のものがビルドされる（外部での書き換えは上書きされることがある点に注意）。

---

## 水を近づけても反応しない / 判定がおかしい

1. **校正したか**：未校正だと判定は不正確。`configValid()` が 0 なら校正ウィザードを実行。
2. **電極の向き**：センス(RA4)とリファレンス(RA0)が**背中合わせ**か。逆や離れすぎは差動が効かない。
3. **接触**：電極がガラス面に密着しているか。浮くと結合が弱まる。
4. **分離不足**：校正で「乾 ≤ 水」や「分離 < 4σ」の警告→結合が弱い。電極接触長を伸ばす／ガラスを薄く／
   [hardware.md](hardware.md) の電極改善（平面電極・ガード）を検討。
5. **生値で確認**：`setOutSel(1)`/`(2)` でセンス/リファレンス生値を見て、水で**センスだけ**動くか確認。
   確認後は `setOutSel(0)` に戻す。

---

## 境界でバタつく（チャタリング）

- **ヒステリシス**を増やす：`setHysteresis()` を大きめに。校正では `hyst = 0.7σ_乾` が自動設定される。
- ゲートを電源周期に合わせる（[usage.md](usage.md) §6）：50Hz=200 / 60Hz≈167。
- 電極結合を強める（gap を広げる）。

---

## 電波（EMI）で誤検出する

- 差動化により **144/435MHz@5W でも 50cm 以上離れていれば正常**（共通モード除去）。
- ただし**近接・高出力の持続 RF はフィルタで除去不可**。対処：
  - **駆動ガード**電極、**同軸/シールド線＋フェライト**、電極リードを短く固定（[hardware.md](hardware.md)）。
  - `setOutSel` でセンス/リファレンス両方が動き**差動値が不動**なら、差動除去は効いている証拠。

---

## 温度表示が数百℃になる

- **異常ではなく未校正**です。`setTempCal()` を電圧に合わせて呼ぶ：
  - 5V：`setTempCal(4.8, 4.0, 94)`
  - 3.3V：`setTempCal(3.3, 2.0, N)`（N を既知温度で再導出。[api.md](api.md) 参照）
- **3.3V なのに 5V 用ファーム（高レンジ）を焼いていないか**も確認（3.3V は低レンジ版が必須）。

---

## 電源投入後に設定が消える / `configValid()==0`

- `save()` を呼んだか（0x0A に 0xA5）。校正後に `pushConfig()`＋`save()` の両方が必要。
- **値空間が変わるファームを焼くと EEPROM magic が変わり旧設定は自動無効化**される仕様
  → 再校正すれば直る（これは意図的な安全機構）。

---

## ビルドできない（PIC ファーム）

- **DFP** `PIC12-16F1xxx_DFP` を入れたか（無いと PIC12F1840 が選べない）。
- XC8 と MPLAB X が入っているか。`.X` プロジェクトを開いてビルド。
- 正常なら Program 約 **1540 words / 37.6%**、RAM **164B / 64%**。

---

## それでも解決しない場合

- `examples/WaterBoaderDemo` を素の構成で焼き、起動時の `resetCause` と `configValid` を確認。
- `recover()` の出力（リセット要因）を見る：`WDT` が立てば「一度ハングして自動復帰」、
  `BOR` なら電圧サグ（電源容量不足を疑う）。
- 厳密な挙動は [docs/SPEC.md](../SPEC.md) の受け入れ基準(§8)・既知の限界(§9) を参照。
